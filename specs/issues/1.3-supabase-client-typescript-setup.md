## Step-by-step Guide

### 1. Install Supabase SDK

Install `@supabase/supabase-js` package:

```bash
npm install @supabase/supabase-js
```

Also install the Supabase CLI for type generation:

```bash
npm install -D supabase
```

---

### 2. Setup environment variables

Create `.env.local` in the project root (same level as `package.json`):

```env
VITE_SUPABASE_URL=https://ebcmjbcklusdjdwaaydg.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key-from-project-settings
```

Replace the values with your actual Supabase project URL and anon key from Task 1.2 (Project Settings → API).

**Note:** The `VITE_` prefix makes these accessible in Vue components via `import.meta.env.VITE_SUPABASE_URL`

---

### 3. Create `src/lib/supabase.ts` client

Create the file `src/lib/supabase.ts` with the following content:

```typescript
import { createClient } from '@supabase/supabase-js'
import type { Database } from './database.types'

const supabaseUrl = import.meta.env.VITE_SUPABASE_URL
const supabaseAnonKey = import.meta.env.VITE_SUPABASE_ANON_KEY

if (!supabaseUrl || !supabaseAnonKey) {
  throw new Error('Missing Supabase environment variables')
}

export const supabase = createClient<Database>(supabaseUrl, supabaseAnonKey)
```

This creates a reusable Supabase client that can be imported throughout the app.

---

### 4. Generate TypeScript types from Supabase schema

The `database.types.ts` file is auto-generated from your Supabase schema. To generate it:

#### Step A: Install Supabase CLI (if not already done)

```bash
npm install -D supabase
```

#### Step B: Link your Supabase project locally

```bash
npx supabase link --project-ref ebcmjbcklusdjdwaaydg
```

When prompted for your Supabase password, use the password you set when creating the project in Task 1.2.

#### Step C: Generate types

```bash
npx supabase gen types typescript --linked > src/lib/database.types.ts
```

This creates `src/lib/database.types.ts` with full TypeScript definitions for all tables (projects, posts, contacts).

**Alternative (without CLI):** You can manually download types from Supabase dashboard:
1. Go to **Project Settings → API → TypeScript**
2. Copy the generated code into `src/lib/database.types.ts`

---

### 5. Update `tsconfig.json` to include lib folder

Ensure `src/lib` is recognized by TypeScript. Check `tsconfig.json` has:

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"]
    }
  }
}
```

Or simply verify `src/**/*` is included in `include`:

```json
{
  "include": ["src"]
}
```

---

### 6. Test Supabase connection in a Vue component

Create a test component or add to `src/App.vue` to verify the connection works:

```vue
<script setup lang="ts">
import { ref, onMounted } from 'vue'
import { supabase } from './lib/supabase'

const message = ref('Testing Supabase connection...')

onMounted(async () => {
  try {
    // Test query: fetch one project
    const { data, error } = await supabase
      .from('projects')
      .select('*')
      .limit(1)

    if (error) {
      message.value = `Error: ${error.message}`
    } else {
      message.value = `✓ Connection successful! Projects: ${data?.length || 0}`
    }
  } catch (err) {
    message.value = `Error: ${err instanceof Error ? err.message : 'Unknown error'}`
  }
})
</script>

<template>
  <div>
    <p>{{ message }}</p>
  </div>
</template>
```

Run `npm run dev` and verify:
- No errors in browser console
- Message displays "Connection successful!" or appropriate error message
- (If you've inserted sample data in Task 1.2, project count will show)

### 7. Add `.env.local` to `.gitignore`

Ensure `.gitignore` contains `.env.local` to prevent committing secrets:

```bash
echo ".env.local" >> .gitignore
```

## Notes

- **Environment variables:** The `VITE_` prefix is required for Vite to expose them to the browser. Never commit `.env.local` to git.
- **Type safety:** The generated `database.types.ts` provides full TypeScript autocomplete and validation for all Supabase operations.
- **Error handling:** The client throws if environment variables are missing, failing fast during startup rather than at query time.
- **Testing:** Once this task is complete, all subsequent data-fetching tasks (e.g., fetching projects, posts) can use this client directly.
