## Description

## Step-by-step Guide

**Files to create/modify:**
- `src/pages/Blog.vue` — Add search input UI and update fetching logic.

**Key decisions:**
- **Filtering:** Use Supabase's `.or()` with `.ilike()` for simple text matching across title and excerpt.
- **State Management:** Keep search query in a local ref; reset pagination offset to 0 on every new search to avoid being stuck on an empty high-indexed page.

**Non-obvious snippets:**
```ts
// Example Supabase query with search and pagination
const fetchPosts = async () => {
  let query = supabase
    .from('posts')
    .select('*', { count: 'exact' })
    .eq('published', true);

  if (searchQuery.value) {
    query = query.or(`title.ilike.%${searchQuery.value}%,excerpt.ilike.%${searchQuery.value}%`);
  }

  const { data, count } = await query
    .order('published_at', { ascending: false })
    .range(from, to);
};
```

---

## Notes
- Ensure the search query is URI-encoded if passed through route params (optional, but good for shareable search URLs).
- Performance: `ilike` with leading wildcards (`%...%`) doesn't use standard indexes efficiently, but for a personal blog scale, it is perfectly acceptable.
