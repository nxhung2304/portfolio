## Step-by-step Guide

**Files to create/modify:**
- `src/pages/About.vue` — add Skills section and CV download link after the Timeline section
- `public/cv.pdf` — static CV file (if not using Supabase Storage)

**Key decisions:**
- Keep skills as a hardcoded constant in `About.vue` (no Supabase fetch needed for v1)
- For CV download, prefer the simpler static file approach (`public/cv.pdf` → `<a href="/cv.pdf" download>`) unless the user wants a private/expiring URL via Supabase Storage
- Reuse `useScrollReveal` composable for the new section's fade-in, consistent with Bio, Stats, and Timeline sections

**Non-obvious snippets:**
```ts
// SKILLS constant shape

]
```

## Notes
- `About.vue` already uses three `useScrollReveal` refs (bio, stats, timeline) — add a fourth for the skills section following the same pattern
- If Supabase Storage is chosen for the CV, generate a public URL via `supabase.storage.from('cv').getPublicUrl('cv.pdf')` and store it in a `ref`; avoid storing private signed URLs that expire at build time
- Skills data is static for now — no need to pull from the DB or create a new Supabase table
