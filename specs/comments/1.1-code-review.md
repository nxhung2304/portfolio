### 2. Missing Path Aliases
**File:** [vite.config.ts](vite.config.ts)  
**Status:** CODE STYLE  
**Description:** No `@/` alias configured. Imports like `@/pages/Home.vue` improve readability  
**Fix:** Add `resolve` configuration to `vite.config.ts`:
```typescript
import { fileURLToPath } from 'node:url'
export default defineConfig({
  resolve: {
    alias: {
      '@': fileURLToPath(new URL('./src', import.meta.url))
    }
  }
})
```
**Also update** `tsconfig.json`:
```json
"baseUrl": ".",
"paths": { "@/*": ["./src/*"] }
```

### 3. Tailwind Theme Not Extracted
**File:** [tailwind.config.js](tailwind.config.js)  
**Status:** REFACTOR  
**Description:** Repeated design tokens should be extracted to constants per rules  
**Future:** When adding colors/spacing, define as:
```javascript
const colors = { primary: '#3b82f6', secondary: '#10b981' }
export default { theme: { extend: { colors } } }
```

### 4. Global Styles in Component
**File:** [src/App.vue](src/App.vue#L8-L18)  
**Status:** CODE STYLE  
**Description:** App-level styles in `<style>` block; should move to `style.css` for consistency  
**Fix:** Move to [src/style.css](src/style.css):

```css
#app {

  -webkit-font-smoothing: antialiased;
  -moz-osx-font-smoothing: grayscale;

}
```

### 6. Router Missing Error Handling
**File:** [src/router/index.ts](src/router/index.ts)  
**Status:** ROBUSTNESS

### 7. Direct Route Access in Templates
**Files:** [BlogDetail.vue](src/pages/BlogDetail.vue#L2), [ProjectDetail.vue](src/pages/ProjectDetail.vue#L2)  
**Status:** CODE STYLE  
**Description:** Uses `$route.params.slug` directly; Composition API best practice  
**Fix:** Use `useRoute()` composable:
```vue
<script setup lang="ts">
import { useRoute } from 'vue-router'
const route = useRoute()
const slug = route.params.slug
</script>
<template>
  <h1>Blog {{ slug }}</h1>
</template>
```
